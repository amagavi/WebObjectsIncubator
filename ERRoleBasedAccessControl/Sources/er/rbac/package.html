ERRoleBasedAccessControl

Overview

ERRoleBasedAccessControl is a framework designed to help with user roles and privileges management. The objective is to provide the infrastructure and common business logic for determining if a user has a specific privilege in one of his assigned roles.

A user can be assigned many roles. A role typically has many privileges (aka permissions). You can define roles and privileges any way you want. It is your app. However if you are unsure, here is some advice (note 'advice', not rules!):

  • Many privileges will represent the four common CRUD operations on core EOEntity types. For example, if you have an entity 'Customer' you might define the following 4 privileges: canCreateCustomer, canReadCustomer, canUpdateCustomer and canDeleteCustomer
  • You can define custom privileges too for example canResetPasswords (perhaps to allow a customer service representative to reset users' passwords)
  • Roles should generally correspond to functional roles in an organization, and it might be better to have narrow functional roles defined, even it if means people get multiple roles assigned in the app. it will make it easier later to divide functional responsibilities. Examples of roles might be CustomerServiceRepresentative, CoffeeBoy, Salesperson, InsideSalesperson, OutsideSalesperson, ShippingClerk, PurchasingClerk, PurchasingManager, etc.

This framework provides reusable WOComponents that can be used as conditionals surrounding content on your pages such as action links, partial or even entire page content helping you to ensure that content and functionality of your application is only visible and available to users with appropriate privileges.

Installation and Setup

Include the ERRoleBasedAccessControl framework in your classpath and configure the properties as shown below.

Properties

The properties below allow you to change the behaviour of ERJasperReports:

er.rbac.userEntity (mandatory)
The name of the entity that represents users in your app.
er.rbac.userPrimaryKeyAttribute
The name of the primary key attribute in the app user entity.
er.rbac.rolesRelationshipName
The name of the relationship dynamically created on your user entity at application startup. It defaults to <roles>
